Ship fast, get feedback, iterate. User explicitly wants to get to TestFlight quickly with a usable core loop, not wait for all features to be built.

**Why:** The goal is real user feedback via TestFlight, then push updates via EAS Update (OTA). Getting 10-20 testers on a working core is more valuable than building 10 features before anyone has used it.

**How to apply:** When planning features, always ask "does this block TestFlight or can it come after?" Phase A (core session loop) ships first. Everything else is post-TestFlight unless it's truly blocking.

Also: user thinks in product/vision terms and wants technical execution mapped to that. Don't just list tech tasks — connect them to the UX outcome they enable.
